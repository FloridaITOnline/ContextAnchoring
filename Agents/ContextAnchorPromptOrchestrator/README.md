# 🤖 Context Anchor Prompt Orchestrator (CAPO)
© 2026 Justin Rodriguez | **Version 1.25**

> "The Compiler for the Linguistic Runtime."

**CAPO** is a meta-orchestrator designed to transform unstructured, natural-language prompts into high-fidelity, **Context Anchor Approved Prompts**. It uses inductive inference to bridge the gap between "human vibes" and "deterministic execution."

---

## 🛠️ How It Works

CAPO treats prompt engineering as a **Linguistic Compilation Process**. It follows a multi-stage pipeline:

1.  **Deconstruction (G1):** Analyzes raw text for intents, constraints, and entities.
2.  **Primitive Inference (G2-G5):** Promotes candidates into formal **Anchors**, **Gates**, and **Loops**.
3.  **Flow Synthesis (G6):** Reconstructs the orchestration logic into a DAG (Directed Acyclic Graph).
4.  **Audit & Validation (G9):** Checks the new prompt against the **Context Anchoring Framework Specification**.
5.  **Refinement Loop (L1):** Interacts with the user to resolve high-severity ambiguities.

---

## 🚀 Usage

To use CAPO, provide the contents of `CAPromptOrchestrator.md` to a high-reasoning LLM (GPT-4o, Claude 3.5, etc.) and then provide your raw prompt.

**Example Input:**
> "I want a bot that helps me study for the bar exam. It should give me a question, wait for my answer, tell me if I'm right, and track my score."

**CAPO Output:**
A structured Markdown prompt featuring:
- **A0 (Intent):** Bar Exam Mastery.
- **A2 (State):** Score tracking anchor.
- **G1 (Question Gate):** Bounded reasoning step for generation.
- **L1 (Cycle Loop):** Step-by-step interaction control.

---

## 🔬 Design Philosophy

- **Ambiguity as Data:** CAPO doesn't fail on vague input; it tracks it in `A11_Conversion_Issues` and asks for clarification.
- **Confidence Modeling:** Every conversion includes an **Inference Confidence Report (A8)**, providing a metric for the structural integrity of the output.
- **System Stability:** Built using the very framework it generates, ensuring self-referential consistency.

---

## 📂 Repository Integration

This orchestrator is the primary tool for migrating legacy prompts into the **Context Anchoring** ecosystem. It is the "Bridge" between V1 (Chat) and V2 (Systems).
